## Phase 7 Pre-conditions

One defect was found that is not a Phase 6 blocker but must be fixed before Phase 7 code begins:

**PRE-7-1 — Metric status always GREEN regardless of score.**
`worker/main.go` lines 89-90:
```go
coll.UpdateMetric(fitness.MetricArtifactCorrectness, res.CorrectnessScore, pb.MetricStatus_METRIC_GREEN, "Ratio")
coll.UpdateMetric(fitness.MetricCodeQuality, res.QualityScore, pb.MetricStatus_METRIC_GREEN, "Ratio")
```
When `CorrectnessScore < 0.90` or `QualityScore < 0.85`, the status should be
`METRIC_AMBER` or `METRIC_RED`. Hardcoding GREEN means the escalation mechanism
in the global fitness vector (§5) never fires for Code Assurance findings. Fix: compare
each score against its declared threshold and set status accordingly before calling
`UpdateMetric`.
