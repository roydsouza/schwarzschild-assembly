### Minor note — not a blocker, track in Phase 3

The `Err("Duplicate")` case was removed from the `add_constraint` error match in `register_constraint` because duplicate detection was moved to the early check. If a TOCTOU race occurs (two concurrent registrations of the same ID), the second `add_constraint` call returns `Err("Duplicate")` which now falls into the `else` arm and returns `Rejected` instead of `Duplicate`. In sequential usage this path is unreachable. Under concurrent access it would return the wrong variant. This is not a Phase 2 blocker — constraint registration is not a concurrent hot path — but the correct fix is to re-add the `Duplicate` arm back to the `add_constraint` error match as a safety net. Address in Phase 3 cleanup.
