## Pre-condition Work Assessment

**Accepted:**

- **MCP host** (`internal/mcp/`): JSON-RPC dispatch (`initialize`, `tools/list`, `tools/call`), `get_analyst_verdict` and `approve_action` tool handlers, HTTP transport on `:8082`, wired in `main.go` at lines 151–156. The tool handler calls `h.spine.ApproveAction` and `h.spine.ReadAnalystVerdict` directly against the `pb.OrchestratorServer` interface — correct pattern, avoids network hop. ✓

- **UNSAFE Merkle leaf** (`server.go` lines 211–218): Leaf is constructed and appended for the UNSAFE branch. Error is logged. Not returning `codes.Internal` on UNSAFE Merkle failure is a defensible design choice — `stream.Send(event)` and `hub.Broadcast(event)` have already executed, so the factory has received the UNSAFE verdict. Aborting with `codes.Internal` here would cause the factory to see an RPC error despite a valid (unsafe) evaluation. The log-and-continue behavior is acceptable here. ✓

- **`go mod tidy`**: `github.com/improbable-eng/grpc-web` and `github.com/rs/cors` are now direct imports (no `// indirect`). ✓

**Advisory (non-blocking for Phase 5 kickoff):**

- **stdio transport missing**: CLAUDE.md Phase 3 spec requires "MCP host with stdio (local) and Streamable HTTP (remote) transports." Only HTTP POST is implemented. The comment in `transport_http.go` line 27 reads "In a full implementation, this would handle SSE for the message stream" — this violates Anti-Slop Rule 1 (no TODO comment without a filed proposal). File a proposal in `proposals/pending/` before Phase 5 ends. The HTTP POST transport is sufficient for unary MCP tool calls; `SubmitProposal`'s streaming response path will need SSE or a different mechanism, but this does not block factory scaffolding.

- **`approve_action` MCP tool bypasses the Translucent Gate signature requirement**: `callApproveAction` calls `h.spine.ApproveAction` with a caller-supplied `signature` string from MCP tool arguments. This means an MCP client can approve a security-adjacent proposal by passing any arbitrary string as `signature`. The Translucent Gate's safety invariant (human must explicitly check a signature box) can be bypassed over MCP. This is a security gap that needs a proposal before Phase 5 ends — the MCP `approve_action` tool should either be removed or restricted to non-security-adjacent proposals, with security-adjacent approvals requiring the Control Panel path only.
