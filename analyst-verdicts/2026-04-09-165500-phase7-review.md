## Required Changes

- [ ] **CRITICAL-1 — `AdvanceLifecycle` has no gate enforcement.**
  CLAUDE.md §3 Phase 7 states explicitly: "Each state transition is gated. No state may be
  skipped. The `AdvanceLifecycle` RPC enforces the gates server-side." The current implementation
  (server.go:503-516) calls `s.store.UpdateAssemblyLineState` unconditionally — any caller can
  advance to any state with any justification string, skip states entirely, or transition backwards.
  This makes the lifecycle a state tracker, not a quality gate.

  Required enforcement (minimum — implement all three):

  1. **No backward transitions.** Look up the current state; if `req.TargetState` is earlier in
     the sequence than the current state, return `codes.FailedPrecondition`.

  2. **No state skipping.** The target state must be exactly one step forward from the current
     state. If it is two or more steps ahead, return `codes.FailedPrecondition`.

  3. **INTAKE → DESIGN gate.** Before advancing to DESIGN, verify that the associated
     `SpecDocument.is_finalized == true` in the DB. If not, return `codes.FailedPrecondition`
     with message "spec must be finalized via finalize_spec before entering DESIGN". This
     corresponds to Roy calling `finalize_spec` in the INTAKE dialog.

  The full gate table (DESIGN→SCAFFOLD requiring Analyst verdict, etc.) requires cross-referencing
  the verdict store and Safety Rail — those can remain as `[PHASE-8]` stubs with a
  `codes.Unimplemented` return so the gate is at least not silently bypassed.

- [ ] **SIGNIFICANT-2 — `set_deployment_target` MCP tool missing.**
  CLAUDE.md §3 Phase 7 lists 6 Requirements Advisor tools. Five are present in
  `root-spine/internal/mcp/server.go`. The sixth — `set_deployment_target(spec_id, target, config)` —
  is absent. `target` must be one of `LOCAL | CONTAINER | AWS | GCP`. The Scaffold Engine uses
  this to select the correct template directory and deployment configuration. Add it to the tool
  list and implement the handler (persisting the target to the SpecDocument row).

- [ ] **SIGNIFICANT-3 — Domain fitness metrics incomplete; threshold wrong.**
  CLAUDE.md §3 Phase 7 specifies 4 domain fitness metrics for the Scaffold Engine. Only 2 are
  declared. Add the missing two:
  - `acceptance_criterion_coverage` — % of spec criteria with non-stub test implementations at
    VERIFY phase. Unit: percent. Higher is better. Escalation threshold: < 100%.
  - `rework_rate` — % of assembly lines requiring > 2 CONDITIONAL verdicts before BUILD.
    Unit: percent. Lower is better. Escalation threshold: > 30%.

  Additionally, `scaffold_success_rate` uses threshold 0.85. Per the spec, escalation fires at
  < 60% (not < 85%). Fix the `EscalationThreshold` to `60.0`.

- [ ] **SIGNIFICANT-4 — `analyst-briefing/` contains only `.gitkeep`.**
  Same issue as code-assurance round 1. The directory exists (anatomy check passes) but has no
  `template.md`. Replace `.gitkeep` with `template.md` containing the standard Phase 7 briefing
  structure.

- [ ] **SIGNIFICANT-5 — No tests in scaffold-engine factory.**
  Anti-Slop Rule 3 is absolute. Required minimum:
  - A test for the `MetricsManager.Register` call (mock the gRPC client; verify the correct
    metric IDs and thresholds are sent).
  - A test for the `AdvanceLifecycle` gate logic once CRITICAL-1 is implemented — verify that
    a backward transition and a state-skip both return `FailedPrecondition`.
  The second test must live in `root-spine/` (internal package), not in the factory. Factory
  tests go in `factories/scaffold-engine/`.

## Safety Rail Implications

`AdvanceLifecycle` without gate enforcement means the SCAFFOLD→BUILD gate (which should require
Safety Rail verification of scaffold artifacts) is currently bypassable. This is not a Safety Rail
trait contract violation — the contract is intact — but it is a gap in the pipeline that Phase 8
must close. The `[PHASE-8]` stubs in `UpdateSkill` and the lifecycle handler are acceptable
placeholders provided they return `codes.Unimplemented` rather than silently succeeding.
