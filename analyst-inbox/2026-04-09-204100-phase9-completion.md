## Summary of Changes

Phase 9 implements the autonomous self-optimization loop for the Prolog Substrate, allowing agents to monitor their own skills, propose optimizations, and verify them in a sandbox before commitment.

### Core Introspection (`meta/introspect.pl`)
- **`inspect_predicate/2`**: Retrieves current predicate definitions.
- **`measure_performance/3`**: Measures predicate latency using local sampling (Phase 1).
- **`test_predicate/3`**: Runs predicates against input sets with `call_with_time_limit/2` for sandboxed evaluation.

### Evolutionary Loop (`meta/improve.pl`)
- Implements the **Observe → Introspect → Construct → Evaluate → Propose** loop.
- **Baseline Strategy**: Implements "Cut Injection" (`!`) to optimize deterministic predicates by stopping early once a solution is found.
- Integrates with `safe_assert/1` for verified, audited updates.

### Safety & Invariants (`meta/verify.pl` & `policies/constraints.chr`)
- **`check_invariants/1`**: Prevents redefinition of critical safety primitives (`safe_assert`, `merkle_commit`, etc.).
- **CHR Hardening**: Added `head_is_meta/1` constraint to `constraints.chr`, throwing a `safety_violation` if an agent attempts to redefine the meta-improvement logic itself.

### Verification Harness
- **`tests/test_introspect.pl`**: Verifies definition reading and goal construction.
- **`tests/test_meta.pl`**: Verifies end-to-end optimization of a "slow" predicate.
- **`tests/test_regression.pl`**: Verifies parity against golden test sets.
