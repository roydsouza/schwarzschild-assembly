### Step 2: Z3 initial policy set (`tier1/z3_policy.rs`)

Register 4 mandatory initial constraints at `Tier1SafetyRail::new()`:

**Constraint 1 — safety_no_self_modify_safety_rail:**
```
category: SafetyCompliance, severity: Mandatory
assertion (SMT-LIB2):
  (assert (=> (= target_path "safety-rail/") (= is_verified true)))
justification: "No change to safety-rail/ may be executed unless it has passed verification"
```

**Constraint 2 — audit_no_merkle_deletion:**
```
category: AuditIntegrity, severity: Mandatory
assertion (SMT-LIB2):
  (assert (=> (= operation_type "delete") (not (= target_component "merkle-log"))))
justification: "Merkle log entries are append-only; no deletion operations are permitted"
```

**Constraint 3 — dhamma_score_minimum:**
```
category: DhammaAlignment, severity: Mandatory
assertion (SMT-LIB2):
  (assert (>= dhamma_score 0.0))
justification: "All proposals must receive a Dhamma alignment score before execution"
```

**Constraint 4 — security_no_unverified_proto_change:**
```
category: Security, severity: Mandatory
assertion (SMT-LIB2):
  (assert (=> (= target_path "aethereum-spine/proto/") (= is_security_adjacent true)))
justification: "Proto contract changes are always security-adjacent and require Translucent Gate"
```

**Note on Z3 assertions:** The SMT-LIB2 assertions above are logical specifications.
The actual Z3 Rust bindings use the `z3` crate's builder API, not raw SMT-LIB2 strings.
Translate these to `z3::ast::Bool` expressions using the z3 crate API.

### Step 3: Wasmtime sandbox (`tier1/sandbox.rs`)

Implement `execute_sandboxed`:
- Use `wasmtime::Engine`, `wasmtime::Store`, `wasmtime::Module`
- Apply WASI preview 2 (`wasmtime_wasi::preview2`) with a restricted preopened directory
- Hard limits: 256 MiB memory, 5-second epoch timeout
- Return `ExecutionResult::Failure { error_kind: ExecutionErrorKind::StaleProof }` if
  `artifact.policy_fingerprint.digest != self.policy_fingerprint().digest`

### Step 4: Tests

Every public function must have a test that catches a real regression.
Use the contract compliance helpers in `safety-rail::contract_tests` as a starting point.

Required test coverage:

- `test_sandbox_timeout_enforced`

### Step 5: OTel instrumentation

Every code path that touches the fitness vector must emit an OTel metric.
Do not use `log::info!` or `println!` on production paths — emit structured OTel events.

Required OTel emissions:
- `aethereum_spine.safety.violations_total` (counter) — incremented on `SafetyVerdict::Unsafe`
- `aethereum_spine.safety.verifications_total` (counter) — incremented on every `verify_proposal`
- `aethereum_spine.safety.verification_duration_ms` (histogram) — elapsed ms for each verification
- `aethereum_spine.safety.constraints_total` (gauge) — count of registered constraints

Use `opentelemetry` and `opentelemetry-otlp` crates (add to `Cargo.toml` with pinned versions
and one-line justification comments).

## Anti-Slop Rules (enforced on review)

1. **No `TODO` without a proposal file.** Format: `// TODO(proposals/pending/filename.md): reason`
2. **No untracked dependencies.** Every new dep in `Cargo.toml` needs a pinned version + comment.
3. **No untested interfaces.** Every public interface has a test catching a real regression.
4. **No silent failures.** Every error path emits an OTel event before returning.
5. **No freeform strings in structured paths.** Use the typed `EventKind` enum from log-schema.json.
6. **No `unsafe` blocks without a `// Safety:` comment** explaining the invariant.
7. **No placeholder implementations.** `todo!()` bodies are veto-eligible.
